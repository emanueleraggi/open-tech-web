A Coffee Landing Pages Showcase Study

March 29th, 2020

Whether we like it or not, landing pages are the most important virtual place to “find out” about a local business, a restaurant or simply looking for the latest feedback about a place. Do you know this restaurant? What do we do? Let’s Google it! Let’s find out whether it is vegan or vegetarian or both. In the new technological era, digitalization plays a very important role. Just think of all the possible reasons why we want to go to a local coffee shop. Studying? Socializing? Simply having a break from the external confusion and enjoy a warm and cozy cup of coffee. And since we are sitting there why not send out a resume to a potential employer!

Let’s look at the business side now. How should a local coffee shop that seeks to increase customer usage of a website to promote sales? It is a great and inexpensive idea. A simple mobile and responsive website should be the first choice for showcasing their creations, food and what their brand means. Here and below some snapshots of a potential landing page I recently published as example:

This is a single landing page layout that I designed with different sections and a variety of functionalities such as a show or hide sidebar, responsive buttons and google map embedded frame. This coffee landing page is just an example of what could be achieved for a customer in relatively small time as an initial prototype. Prototyping is very important as it gives a sense of what the business should communicate and what should be communicated.

For this landing page and its prototyping phase it used HTML, CSS, and JavaScript to quickly create the layout of the website. The website follows a responsible design approach and can be easily shown on a smartphone, a tablet and a regular Desktop computer.

The main slider of the Home page shows a video in the background for the desktop and tablet screen. The shop name is displayed with a stylistic, bold font. The About section features a custom grid display of carefully chosen images and text. To account for mobile devices, the text and images scale according to the width of the page. Hover effects are used to indicate clickable objects and a modal is displayed using JavaScript functionalities in order to enlarge the clicked image. The customer can provide a name and last name for additional perks such as a free coffee, additional discount and much more, depending on the business owner's interests. The name, or nickname, of the customer is then randomly chosen to give these perks.

Some example code snippets for this specific landing pages are shown below. In particular note the design of the toggle button on top of the header, which is a combination of positions absolute and relative and z-index:

html
<!-- video switch -->
<div class="video__switch-container">
  <div class="video__switch">
    <span>on</span>
    <span>off</span>
    <div class="video__switch-btn"></div>
  </div>
</div>
css
/* video switch */
/* We will not display it at the loading stage */
.video__switch-container {
  display: none;
}


/* But after I get to a specific monitor min width the switch should
   become visible */
@media screen and (min-width: 768px) {
  .video__switch-container {
    display: block;
    /* because the header is the *container* holding the switch
    (plus other components) the position of the switch
    is absolute with respect to the header */
    position: absolute;
    bottom: 5%;
    right: 5%;
    z-index: 3;
  }
  .video__switch {
    position: relative;
    width: 5rem;
    background: var(--yellow);
    text-transform: uppercase;
    display: flex;
    justify-content: space-between;
    padding: 0.25rem;
    cursor: pointer;
    border-radius: 0.5rem;
  }
  .video__switch-btn {
    position: absolute;
    top: 0;
    left: 0;
    background: var(--brown);
    height: 100%;
    width: 50%;
    border-radius: 0.5rem;
    transition: all 0.3s ease-out;
  }
  .btnSlide {
    left: 50%;
  }
}

Finally below a JavaScript piece of the toggle button and how all the event listeners are set:

js
eventListeners()


function eventListeners() {
  const ui = new UI()


  // video control
  document
    .querySelector('.video__switch')
    .addEventListener('click', function () {
      ui.videoControls()
    })
}


function UI() {}


// play/pause the video
UI.prototype.videoControls = function () {
  // document.querySelector('.video__switch-btn').classList.toggle('btnSlide');
  let btn = document.querySelector('.video__switch-btn')
  // if it does not have the class, then lets add it
  if (!btn.classList.contains('btnSlide')) {
    btn.classList.add('btnSlide')
    document.querySelector('.video__item').pause()
  }
  // else is it is already clicked lets remove the class
  else {
    btn.classList.remove('btnSlide')
    document.querySelector('.video__item').play()
  }
}
