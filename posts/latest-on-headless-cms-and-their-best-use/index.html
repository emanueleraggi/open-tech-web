Latest On Headless CMS And Their Best Use

August 15th, 2020

An Application Programming Interface, also known as API, is a software intermediary between two or more devices to mutually communicate. For example every time a user accesses its own Google Drive, they are using an API to do that. Are you watching the weather forecast or simply dropping a message on Instagram? Well, you are using an API. Do you like to visualize restaurant icons on your google map you are watching on your smartphone? Well you are using an API to see that too.

It is easier than ever in 2021 to build a website in a relatively small period of time. Latest technology allows one or two days just to put together a quick design with very simple components and one single landing page to one or two weeks for a detailed website. Not to mention that together with paying options there are the open source projects for prototyping and wireframing such as the Pencil Project. Of course this has less features than the most advanced, but still is comparable because of easiness to use and basic components. It allows you to import images, create your personal collection of components, icons and images and has all the features a typical prototyping system has. Pencil allows to save the draft as an HTML interactive project, meaning that right after the prototype is ready and downloaded as a pdf document, it is still possible to pass around the mock wireframe and interact with clickable buttons that bring you to the next page or feature.

Great focus should also be connected to content. In this episode we will talk about the emerging free open-source CMS which is 100% JavaScript: Strapi. A headless Content Management System (CMS) is a software that allows the creation of content to be then deployed in a production environment. Two principal distinctions are the foundation of a CMS such as: a) enterprise content management and 2) web content management. In the first case there can be multiple users working on a collaborative space, whereas in the second case there are typically single authors for websites including management of photos, videos, audio, maps etc.

When in a production environment a user should remember to create the production (or build) folder with Strap running in the background, otherwise the hosting platform will incur into potential compilation errors. Other than that Strapi has an excellent content user interface that allows a user to add, modify or remove content from a website without contracting an expensive professional to produce modifications. The fact that it is easy to use allows users with limited experience to be comfortable about it right from the beginning. Below a proposal of a resume website that uses Strapi as a headless CMS.

The website is composed and divided into five pages: a Home page that describes the Services, Experience and Latest Blogs Session, About page with a brief description of the developer, a Projects page that lists all the project built or in-progress, a Blog page that collects all the posts a user has been publishing and a Contact page that carries a simple but elegant form for contact submission. Each session has a responsive button that brings the user to the prosper page. Strapi has the great features of having reusable components. For example the Experience section is composed of a title, a category, a date and bulleted points. This specific structure was set once on Strapi and then simply replicated, cutting the development time in favor of content creation paying particular attention to SEO. Remember that now more than ever and as previously mentioned, users find businesses online introducing specific keywords. So the biggest the efforts in the content creation the higher the possibility to be found. This website was built using the popular Gatsby framework with the Strapi CMS.

For a list of popular headless CMS there will be another episode, so stay tuned.
