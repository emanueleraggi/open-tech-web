A Car Dealership Landing Pages Showcase Study

June 15th, 2020

We are almost finished for our landing page showcase and today the attention is focused on a small car dealership business. A consequence of this pandemic is that the workforce diminished and consequently the number of people traveling to work every day diminished too. The digital era is taking over people’s lives and the always more common Zoom meetings are filling up our lives at an unprecedented pace. Differently to our European folks, who are struggling more about smart working due to different work ethic and regulations, here in the US this is now taking up. Less cars are being sold because less people need to travel. Why go to a meeting when we have online tools? Well, it depends on the business because not always “smart working” is beneficial. Would you buy a car online without even trying it? Below it was created a quick prototype of a car dealership landing page in an effort to show how impactful this can be and at a very inexpensive cost for the customer. Also the same live website can be found here for reference

The website is light and presents itself well. Interacting with the navbar you can jump between different sections of the page. A display of what is available for selling is also shown and zooming capabilities are activated to a better user experience or technically speaking, UX. The About us section and the contact section are simple and intuitive; it is possible to directly contact the dealer for the latest available models or for simple inquiries. The capabilities of the dealership are clearly shown in the website. Clarity of what can and can’t be done are present. It did not use any framework such as Gatsby, Angular or React, but simply HTML for designing the user interface, CSS for styling it and JavaScript to give some logic. It is important to stress that these types of static websites can be greatly leveraged as landing pages and can be as complex as deemed to be. However the bottom line remains, with a simple and small but professional landing page it is possible to save money and still be able to reach a great and good-looking result. In this sample page we can see how the navbar is divided and is responsive; it is possible to jump in any part of the page by just a simple button click. Images are carefully chosen and logic of the website makes the page very intuitive and easy to use, which is what a customer would want: ease of use and communicating the point straightforward, which is the mission of the business.

Two aspects are worth mentioning in this post: a) the movement of th backgorund which is zoomin inn and out on a regular interval and 2) the search button for specific car types such as Amercian, German etc. This last point is mainly guided by the JavaScript logic behind it and filtering the search for our specific needs. As it is shown below in the css the animation is achieved using the parameter animation-name: zoom and giving it a name. This name is then associate to a transformation function that is assigned to a specific keyframes opration as demonstarted below:

css
/* header */
.header {
  clip-path: polygon(0 0, 100% 0, 100% 95%, 50% 100%, 0 95%);
  color: var(--mainWhite);
  overflow: hidden;
}
.height-max {
  min-height: calc(100vh - 76px);
  background: url('../img/headerBcg.jpeg');
  background-position: center;
  background-size: cover;
  background-attachment: fixed;
  background-repeat: no-repeat;
  animation-name: zoom;
  animation-duration: 20s;
  animation-delay: 5s;
  animation-iteration-count: infinite;
}
@keyframes zoom {
  0% {
    transform: scale(1);
  }
  50% {
    transform: scale(1.2) translateX(-5%);
  }
  100% {
    transform: scale(1);
  }
}

Finally using the filtering function available in JavaScript we proceed with selecting specific brand of cars after writing a specific function that does the job we are looking for:

js
const FilterCars = (() => {
  const filterBtn = document.querySelectorAll('.filter-btn')
  // console.log(filterBtn);
  filterBtn.forEach(btn => {
    // we want to grab the (event) because in the html
    // we set the class of data-filter="all", data-filter="american" etc...
    btn.addEventListener('click', event => {
      // lets get the value we have in the filter
      const value = event.target.dataset.filter
      // console.log(value);
      const singleCar = document.querySelectorAll('.single-car')
      // Now after selecting all the cars, lets loop all the cars
      singleCar.forEach(car => {
        if (value === 'all') {
          car.style.display = 'block'
        } else {
          !car.classList.contains(value)
            ? (car.style.display = 'none')
            : (car.style.display = 'block')
        }
      })
    })
  })
})()
